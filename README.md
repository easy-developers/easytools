# easytools

java web工程的最佳实践

# 为什么要创建这个项目

很多大公司面试动不动就问领域驱动设计，但是实际上在开发项目的时候连控制层、服务层、持久化层的职责都没搞明白。 这个项目核心就是想把这三层彻底的写明白了，写一个java web工程的最佳实践。

# 什么情况下要使用这个项目

* 工程分层不明确
* 每个人代码风格不统一 经常扯皮

# 使用本项目的好处

* 每个人写出一样的代码 非常容易看懂别人的代码
* 不用看实现就敢用别人的方法
